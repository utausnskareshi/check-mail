# CheckMail - Windows 11 メール受信チェッカー

複数のメールアカウント(IMAP/POP3)を定期的にチェックし、新着メールがあれば Windows 11 のタスクバー(システムトレイ)にバッジ付きアイコンで通知する Windows デスクトップアプリケーションです。

## 1. 機能概要

- **複数メールアカウント対応**  
  IMAP および POP3 プロトコルに対応。個人用・会社用など、複数のメールアカウントを同時に監視できます。

- **システムトレイ常駐**  
  起動するとタスクバーの通知領域(システムトレイ)にアイコンが表示されます。ウィンドウを閉じてもバックグラウンドで動作し続けます。

- **未読数バッジ表示**  
  新着メールがある場合、トレイアイコンに赤いバッジ(数字)が表示されます。一目で未読メールの存在と件数を確認できます。

- **定期自動チェック**  
  設定した間隔(分単位)で自動的にメールサーバーに接続し、新着メールの有無をチェックします。

- **Windows通知**  
  新着メール検出時に Windows のバルーン通知(トースト通知)を表示します。

- **パスワード暗号化**  
  メールパスワードは Windows DPAPI (Data Protection API) で暗号化して保存されます。平文でファイルに保存されることはありません。

- **Windows起動時の自動起動**  
  設定により、Windows 起動時に自動的にアプリを起動できます。

- **シングルインスタンス**  
  アプリの二重起動を防止します。

## 2. 動作環境

- OS: Windows 10 / Windows 11 (64bit)
- 自己完結型のため、.NET ランタイムのインストールは不要です。

## 3. ビルド方法

ソースコードからビルドする場合は、以下の手順を実行してください。

### 前提条件

- .NET 10 SDK がインストールされていること

### ビルド手順

```bash
# (1) プロジェクトフォルダに移動
cd <任意のフォルダ>\CheckMail

# (2) Release ビルド(自己完結型・単一ファイル)
dotnet publish -c Release
```

生成される実行ファイルの場所:

```
<任意のフォルダ>\CheckMail\bin\Release\net10.0-windows\win-x64\publish\CheckMail.exe
```

> 自己完結型のため、.NET ランタイムが含まれた exe が生成されます(約 170MB)。このexe を任意の場所にコピーして使用できます。

## 4. 起動方法

### 方法1: ビルド済みexeから起動

以下のパスの `CheckMail.exe` をダブルクリックして起動します。

```
<任意のフォルダ>\CheckMail\bin\Release\net10.0-windows\win-x64\publish\CheckMail.exe
```

### 方法2: 開発時にソースから直接起動

```bash
cd <任意のフォルダ>\CheckMail
dotnet run
```

### 起動後の動作

- 起動すると、タスクバーの通知領域(システムトレイ)に封筒アイコンが表示されます。
- メインウィンドウは表示されません(トレイ常駐モード)。
- 初回起動時はメールアカウントが未登録のため、トレイアイコンを右クリックして「設定」を選択し、メールアカウントを登録してください。

## 5. 初期設定(メールアカウントの登録)

1. システムトレイの封筒アイコンを右クリック
2. メニューから「設定」を選択
3. 設定画面が表示される
4. 左下の「＋」ボタンをクリックしてアカウントを追加
5. 右側のフォームに以下の情報を入力:

   | 項目 | 説明 |
   |------|------|
   | 表示名 | わかりやすい名前(例: 会社メール、個人Gmail) |
   | メールアドレス | user@example.com |
   | サーバー | メールサーバーのホスト名(例: imap.gmail.com) |
   | ポート | サーバーのポート番号(プロトコル変更時に自動設定) |
   | プロトコル | IMAP または POP3 |
   | SSL/TLS | SSL/TLS接続を使用するか(通常はチェック) |
   | ユーザー名 | メールサーバーの認証ユーザー名 |
   | パスワード | メールサーバーの認証パスワード |

6. 「接続テスト」ボタンで接続確認
   - 成功すると未読メール数が表示されます。
   - 失敗するとエラーメッセージが表示されます。
7. 「保存」ボタンでアカウント設定を一時保存
8. 必要に応じて一般設定を変更:
   - **チェック間隔**: メールを自動チェックする間隔(分単位、最小1分)
   - **Windows起動時に自動起動**: チェックするとPC起動時に自動で起動
   - **通知を表示**: 新着メール検出時にWindows通知を表示
9. 「全て保存して閉じる」で設定を保存

### 主要メールサービスの設定例

#### Gmail (IMAP)

| 項目 | 値 |
|------|-----|
| サーバー | imap.gmail.com |
| ポート | 993 |
| SSL | 有効 |

> Gmailの場合、「アプリパスワード」の生成が必要です。  
> Googleアカウント → セキュリティ → 2段階認証 → アプリパスワード

#### Outlook.com / Microsoft 365 (IMAP)

| 項目 | 値 |
|------|-----|
| サーバー | outlook.office365.com |
| ポート | 993 |
| SSL | 有効 |

#### Yahoo!メール (IMAP)

| 項目 | 値 |
|------|-----|
| サーバー | imap.mail.yahoo.co.jp |
| ポート | 993 |
| SSL | 有効 |

## 6. 基本的な使い方

### システムトレイアイコンの操作

- **ダブルクリック**: メインウィンドウを表示
- **右クリック**: コンテキストメニューを表示
  - 「表示」 → メインウィンドウを表示
  - 「今すぐチェック」 → 即座にメールチェックを実行
  - 「設定」 → 設定画面を表示
  - 「終了」 → アプリケーションを完全に終了

### アイコンの見方

| アイコン状態 | 意味 |
|------------|------|
| 青い封筒アイコン(バッジなし) | 新着メールなし |
| 封筒アイコン + 赤い丸に数字 | 未読メールあり(数字は合計未読数) |
| ツールチップ(マウスホバー) | 「CheckMail - 未読: X通」と表示 |

### メインウィンドウ

- 各アカウントの未読数、メールアドレス、最終チェック時刻を一覧表示
- エラーが発生したアカウントは赤文字でエラーメッセージを表示
- 「今すぐチェック」ボタン: 手動で即座にメールチェック
- 「設定」ボタン: 設定画面を表示
- ×ボタン(閉じる): ウィンドウを非表示にする(アプリは終了しない)
- 最小化: ウィンドウを非表示にする

> アプリを完全に終了するには、トレイアイコン右クリック → 「終了」を使用してください。

## 7. 設定画面の詳細

### アカウント管理

- 左側のリストに登録済みアカウントが表示されます。
- 「＋」ボタン: 新しいアカウントを追加
- 「－」ボタン: 選択中のアカウントを削除(確認ダイアログあり)
- アカウントを選択すると、右側に詳細設定が表示されます。

### プロトコルの違い

#### IMAP (推奨)

- サーバー側で未読/既読を管理するため、正確な未読数を取得できます。
- 複数のデバイスで同じメールアカウントを使用している場合に適しています。
- デフォルトポート: SSL=993, 非SSL=143

#### POP3

- POP3プロトコルには未読/既読の概念がないため、サーバー上のメッセージ総数を「未読数」として表示します。
- デフォルトポート: SSL=995, 非SSL=110

### 一般設定

- **チェック間隔**: 1分〜任意の間隔で設定可能(デフォルト: 5分)
- **Windows起動時に自動起動**: レジストリに登録されます
- **通知を表示**: 新着メール検出時のバルーン通知の有効/無効

## 9. 設定ファイルの保存場所

設定ファイルは以下のパスに保存されます:

```
%APPDATA%\CheckMail\settings.json
```

通常は以下のようなパスになります:

```
C:\Users\<ユーザー名>\AppData\Roaming\CheckMail\settings.json
```

設定ファイルにはメールアカウント情報(サーバー、ユーザー名、暗号化済みパスワード)やチェック間隔などが JSON 形式で保存されます。

> パスワードはDPAPIで暗号化されており、平文では保存されません。  
> 設定をリセットしたい場合は、上記ファイルを削除してアプリを再起動してください。

## 10. セキュリティについて

### パスワードの暗号化

メールパスワードは Windows DPAPI (Data Protection API) を使用して暗号化されます。

- 暗号化は現在のWindowsユーザーのスコープで行われます。
- 同じWindowsユーザーアカウントでのみ復号化が可能です。
- 設定ファイルを別のPCや別のユーザーにコピーしても、パスワードは復号化できません。
- 暗号化されたパスワードはBase64エンコードされてJSONに保存されます。

### 注意事項

- メールサーバーへの接続にはSSL/TLSの使用を強く推奨します。
- Gmailを使用する場合は、Googleアカウントで「アプリパスワード」を生成して使用してください(通常のパスワードでは接続できません)。
- セキュリティ上の理由から、設定ファイルを他者と共有しないでください。

## 11. トラブルシューティング

### アプリが起動しない

**「CheckMailは既に起動しています」と表示される場合:**  
タスクバーの通知領域(隠れているインジケーター含む)を確認してください。既に起動中のインスタンスがあります。

### メールサーバーに接続できない

- サーバー名、ポート番号、SSL設定を確認してください。
- ユーザー名とパスワードが正しいか確認してください。
- ファイアウォールやセキュリティソフトがポートをブロックしていないか確認してください。
- Gmailの場合: アプリパスワードを使用しているか確認してください。  
  Googleアカウント → セキュリティ → 2段階認証 → アプリパスワード

### 未読数が正しくない

POP3の場合: POP3プロトコルには未読/既読の概念がないため、サーバー上のメッセージ総数が表示されます。正確な未読数が必要な場合はIMAPの使用を推奨します。

### トレイアイコンが表示されない

Windows 11 の設定 → 個人用設定 → タスクバー → その他のシステムトレイアイコン で、CheckMailが非表示になっていないか確認してください。

### 設定をリセットしたい

以下のファイルを削除してアプリを再起動してください:

```
%APPDATA%\CheckMail\settings.json
```

### 自動起動を解除したい

- 設定画面で「Windows起動時に自動起動」のチェックを外して保存してください。
- または、Windowsの「タスクマネージャー」→「スタートアップ」タブで CheckMail を無効にしてください。
